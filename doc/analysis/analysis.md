## 1. 项目概述

### 1.1 项目定位

**Datto Block Driver (dattobd)** 是一个 Linux 内核模块，解决了 Linux 传统快照工具的关键限制：

| 传统工具 (LVM/Device Mapper) | dattobd |
|------------------------------|---------|
| 需要额外的备用卷 | COW 文件存储在源卷本身 |
| 需要卸载文件系统 | 支持在线实时快照 |
| 快照后需要合并数据 | 增量备份，无需合并 |
| 操作复杂，需要停机 | 无需停机，无缝操作 |

### 1.2 核心特性

- **实时快照**: 在块设备级别捕获数据变化
- **Copy-on-Write**: 使用 COW 机制保护原始数据
- **增量备份**: 只传输变化的块
- **跨重启持久化**: 通过 COW 文件保存状态
- **广泛兼容**: 支持 2.6.18 至最新内核

### 3.2 COW 管理器核心数据结构

```c
struct cow_manager {
    struct dattobd_mutable_file *dfilp;  // COW 文件指针
    uint32_t flags;                       // 状态标志
    uint64_t curr_pos;                    // 当前写入位置
    uint64_t data_offset;                 // 数据区起始偏移
    uint64_t file_size;                   // 文件大小
    uint64_t seqid;                       // 快照序列号
    uint8_t uuid[COW_UUID_SIZE];          // UUID
    unsigned long sect_size;              // Section 大小 (4096)
    unsigned long total_sects;            // 总 Section 数量
    unsigned long allowed_sects;          // 允许缓存的 Section 数量
    struct cow_section *sects;            // Section 数组
};

struct cow_section {
    char has_data;         // 是否有数据
    unsigned long usage;   // 使用计数 (LRU)
    uint64_t *mappings;    // 块映射数组
};
```

### 4.3 快照模式下的 BIO 处理

```c
// tracer.c: snap_trace_bio()
static int snap_trace_bio(struct snap_device *dev, struct bio *bio)
{
    // 1. 检查是否需要 COW
    if (!bio_needs_cow(bio, dev->sd_cow_inode))
        return SUBMIT_BIO_REAL(dev, bio);
    
    // 2. 计算对齐的扇区范围
    start_sect = ROUND_DOWN(bio_sector(bio) - dev->sd_sect_off, SECTORS_PER_BLOCK);
    end_sect = ROUND_UP(bio_sector(bio) + bio_size(bio), SECTORS_PER_BLOCK);
    
    // 3. 分配追踪参数
    tp_alloc(dev, bio, &tp);
    
    // 4. 创建读克隆 BIO
    bio_make_read_clone(dev_bioset(dev), tp, bio, start_sect, pages, &new_bio);
    
    // 5. 提交读克隆 (读取原始数据)
    dattobd_submit_bio(new_bio);
    
    // 读克隆完成后，回调函数会将数据写入 COW 文件
    // 然后放行原始写请求
}
```

## 8. 关键技术点

### 8.1 跨内核版本兼容

dattobd 通过特性检测实现跨内核版本兼容：

```
configure-tests/feature-tests/
├── bio_alloc.c           // 检测 bio_alloc() 函数签名
├── blk_alloc_queue_1.c   // 检测 blk_alloc_queue() 参数
├── bdev_open_by_path.c   // 检测新的块设备打开 API
├── ftrace_regs.c         // 检测 ftrace_regs 结构
└── ...
```

genconfig.sh 生成 kernel-config.h，代码中使用条件编译适配不同内核版本：

```c
#ifdef HAVE_BIO_ALLOC
    bio = bio_alloc(GFP_NOIO, nr_vecs);
#else
    bio = bio_alloc_bioset(GFP_NOIO, nr_vecs, &dev->sd_bioset);
#endif
```

### 8.2 内存缓存管理 (LRU 策略)

**缓存结构:**
- `cow_manager.allowed_sects`: 可缓存的最大 Section 数量
- `cow_manager.allocated_sects`: 当前已分配的 Section 数量
- `cow_section.usage`: 使用计数 (用于 LRU)

**缓存清理流程:**
1. 当 `allocated_sects > allowed_sects` 时触发清理
2. `__cow_cleanup_mappings()`:
   - 找到最大 usage 值
   - 计算中位数 `threshold = max_usage / 2`
   - 将 `usage <= threshold` 的 Section 同步到磁盘
   - 释放这些 Section 的内存
   - 重置所有 usage 为 0

**优势:**
- 热点数据保留在内存中
- 冷数据自动换出到磁盘
- 内存使用可控

### 8.3 自动扩展机制

**配置参数:**
- `step_size_mib`: 每次扩展的大小 (MiB)
- `reserved_space_mib`: 保留给用户的空闲空间 (MiB)

**扩展条件:**
1. COW 文件空间不足 (`curr_size >= file_size`)
2. 可用空间足够: `available_blocks >= (step_size + reserved_space)`

**扩展流程:**
```
__cow_write_data()
    │
    ▼ (空间不足)
cow_auto_expand_manager_get_allowance()
    │
    ▼ (允许扩展)
tracer_expand_cow_file_no_check()
    │
    ▼
__cow_expand_datastore()
    │
    ▼
file_allocate() // 扩展文件
```

## 总结

dattobd 是一个设计精良的 Linux 内核模块，通过以下核心技术实现了高效的块设备快照和增量备份：

1. **Copy-on-Write 机制**: 在块级别捕获数据变化，保证数据一致性
2. **I/O 拦截技术**: 使用 ftrace 和 BIO 拦截实现透明的 I/O 追踪
3. **状态机管理**: 完善的状态转换机制支持跨重启持久化
4. **内存缓存**: LRU 策略的 Section 缓存提高性能
5. **跨内核兼容**: 特性检测机制支持广泛的内核版本

该设计使得 dattobd 能够在不停止服务的情况下完成实时快照和增量备份，是企业级备份解决方案的理想选择。
